한글을 지원하지 않는 폰트가 지정되어 있음

config.ts:206 — fonts.families: Inter

본문·제목 폰트가 Inter인데, 이 폰트에는 한글 글리프가 없습니다. 사이트 전체가 한글 콘텐츠인데 폴백 체인에도 한글 폰트가 지정돼 있지 않습니다.

대부분의 Windows/Mac/모바일 방문자는 브라우저가 시스템 한글 폰트로 자동 대체해 정상적으로 보이지만, 이 경우에도 라틴 문자(Inter)와 한글(시스템 폰트)의 글꼴이 섞여 통일감이 떨어집니다. 한글 폰트가 없는 일부 Linux 환경에서는 실제로 글자가 깨져 보일 수 있습니다.

제안 fonts.families에 Pretendard 또는 Noto Sans KR을 추가하고 CDN으로 로드 — 이미 fonts.source: "cdn" 설정이 되어있어 config.ts 몇 줄만 바꾸면 적용됩니다.
테스트 환경(한글 폰트 없는 sandbox)에서 재현된 모습

태그 URL 대소문자 구분

/posts/tag/[...tag].astro

/posts/tag/Spring은 정상 접속되지만 /posts/tag/spring은 404입니다. 사이트 내 태그 클릭 링크는 항상 올바른 대소문자로 생성되어 문제가 없지만, 외부 공유 링크나 직접 입력 시 깨질 수 있습니다.

제안 태그 매칭 시 대소문자를 구분하지 않도록(case-insensitive) 비교 로직 추가.
