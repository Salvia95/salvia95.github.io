## 안녕하세요, 양정훈입니다.

Java와 Spring 기반의 백엔드 시스템을 설계하고 운영하는 소프트웨어 엔지니어입니다.

금융, 공공 등 기술적·제도적 제약이 많은 환경에서 일하며, 특정 기술보다는 **기술적 원리**를 통해 문제를 해결하는 것을 중요하게 생각합니다. 비즈니스와 사용자 행동 분석을 바탕으로 프로세스를 설계하고, 레거시 시스템의 성능을 개선하는 일을 주로 하고 있습니다.

---

## 주로 하는 일

- 모놀리스 아키텍처에 모듈러 아키텍처와 비동기 이벤트 통신을 점진적으로 적용하여 시스템 결합도를 낮추는 작업
- Spring Batch 기반의 비동기 병렬 처리 설계를 통한 대규모 트랜잭션 성능 최적화
- DDD/TDD를 적용한 도메인 모델 분리와 테스트 커버리지 확보
- Oracle, PostgreSQL 환경에서의 데이터 모델링과 쿼리 튜닝

---

## 기술 스택

- **Backend**: Java, Kotlin, Spring Framework, Spring Boot, Spring Batch
- **Database**: Oracle RDBMS, PostgreSQL, JPA
- **Architecture**: 모듈러 모놀리스, DDD, 이벤트 기반 아키텍처
- **Messaging**: Kafka
- **Tools**: Git, Obsidian, GitLab

---

## 이 블로그에 대해

**New Branch!** 는 업무 속에서 겪은 기술적 문제 해결 과정과 회고를 기록하는 공간입니다.

실패와 시행착오를 솔직하게 공유하며, 그 과정에서 얻은 인사이트를 정리합니다. 도메인 지식의 유실을 방지하고, 같은 문제를 고민하는 누군가에게 작은 단서가 되기를 바랍니다.
